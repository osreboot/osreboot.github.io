"Iota Nor" Puzzle Game link

 For-Fun | Solo Project | 2023

C++

OpenGL 4

GLSL

GLFW

A refined version of my old LD38 game jam submission that features an experimental glass tile shader, graphical improvements, and a mechanics overhaul.

This game does not use ray-tracing. In fact, it doesn't even use 3D graphics! To achieve the tile lighting, "refraction map" UV lookup textures were pre-baked in Blender. The GLSL fragment shader samples nine refraction maps (representing adjacent tiles) for every glass pixel and determines which background colors to propagate.

While this could have easily been done with ray-tracing, I wanted to see if I could still achieve a convincing effect without relying on specialized hardware!

"Osmium" Game Prototype

For-Fun | Solo Project | 2022-2023

Java

OpenGL

GLSL

GLFW

LWJGL 3

A realtime multiplayer 2D space combat simulator based on Newtonian physics. Featuring ship trajectory projection and maneuver planning, real-life missile tracking algorithms (proportional navigation), and automated turrets capable of leading target velocity and acceleration!

A modified ECS architecture allows for a clean implementation of rollback netcode (server-client synchronization with client prediction and correction). The project is built directly with pure Java sockets (TCP) and does not rely on helper libraries for networking. Deterministic game physics enables input-only communication and allows for hundreds of unique bullet and missile entities at no additional networking expense.

"Nucleocide" Game Prototype link

 For-Fun | 5-Person Team Project | 2021

Java

OpenGL

GLSL

LWJGL 2

Nucleocide is a simple multiplayer top-down shooter concept.

My primary role was to construct the game's rendering system. Built using a Java-based OpenGL binding (LWJGL), the system features dynamic pixel-perfect shadows and intricate world material lighting.

A deferred rendering architecture simplifies the code footprint and allows both entities and tiles to exhibit a range of lighting effects. World tile visual data is supplied in the form of color, normal, metalness, and occlusion textures. Dynamic entities can supply additional emission and displacement textures. A custom GLSL fragment shader ingests this data over several rendering passes and combines it to produce the final image.
